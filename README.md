# pixel-factory-to-ota

pixel-factory-to-ota is a script that converts Google Pixel factory images to an OTA. This is useful for folks who run their own custom-signed versions of the Pixel stock OS, but want to use Android versions where Google does not provide full OTAs, such as Android Canary.

OTAs generated by this script cannot be flashed as-is and _must_ be re-signed with avbroot.

### Usage

1. Install [uv](https://docs.astral.sh/uv/) or manually set up a Python virtualenv.

2. Ensure that the [afsr](https://github.com/chenxiaolong/afsr) and [avbroot](https://github.com/chenxiaolong/avbroot) executables are present in `PATH`.

3. Download a factory image. The URLs for unpublished factory images (eg. Android Canary) can be found using [pixel-factory-images](https://github.com/chenxiaolong/pixel-factory-images).

4. Generate an OTA from a factory image:

    ```bash
    uv run pixel-factory-to-ota.py -i factory.zip -o ota.zip
    ```

    Note that this requires a lot of disk space in the system temp directory. If there is not enough space, set the `TMPDIR` (Linux, Mac) or `TEMP` (Windows) environment variable to a different directory.

5. Patch the resulting OTA with avbroot.
